# [C4] 120 Million I/O Per Second with a Standard 2U Intel Xeon System

- **학회/연도:** SPDK Blog, 2023
- **저자:** Intel SPDK Team
- **분류:** CPU Initiated I/O

## 핵심 요약 (1~2문장)
표준 2U Intel Xeon 서버에서 SPDK NVMe 드라이버를 활용하여 **시스템 전체 120M IOPS** 달성. 단일 CPU 코어로 **10M+ IOPS**를 처리할 수 있음을 실증하여, CPU Initiated I/O의 현재 기술적 상한선을 보여줌.

## 읽기 전 질문
- 단일 코어 10M IOPS를 달성하기 위해 어떤 소프트웨어/하드웨어 최적화가 적용되었는가?
- 120M IOPS 달성 시 CPU 코어 수 대비 선형 스케일링이 가능한가?
- 이 결과를 실제 스토리지 애플리케이션에 적용할 때의 현실적 제약은 무엇인가?

## 주요 내용 정리

### 1. 문제 정의 (Problem)
- NVMe SSD 개당 수백만 IOPS가 가능하지만, **소프트웨어 스택이 병목**
- 기존 Linux 커널 NVMe 드라이버: 코어당 수십만~백만 IOPS 수준에서 CPU 포화
- **질문:** "표준 서버 하드웨어에서 소프트웨어 최적화만으로 IOPS를 얼마까지 끌어올릴 수 있는가?"
- SPDK의 코어당 IOPS 효율과 멀티코어 스케일링 한계를 탐구
- 이를 통해 CPU Initiated I/O 방식의 이론적/실용적 상한을 제시

### 2. 제안 방법 (Approach)
- **SPDK NVMe 드라이버** 기반 성능 극대화
- **하드웨어 구성:**
  - 표준 2U Intel Xeon 서버 (3세대 또는 4세대 Xeon Scalable)
  - 다수의 NVMe SSD (24개 이상) 장착
  - PCIe Gen4/Gen5 슬롯 활용
- **소프트웨어 최적화:**
  - SPDK perf 도구 (`spdk_perf`) 활용
  - CPU 코어 핀닝 (각 코어가 특정 NVMe 큐 전담)
  - NUMA 친화적 I/O 배치
  - 인터럽트 완전 제거 (순수 폴링)
  - hugepages를 통한 TLB miss 최소화
  - 불필요한 데이터 복사 제거 (zero-copy)
- **측정 방법:** 4KB random read 기준 IOPS 측정, 코어 수를 점진적으로 증가시키며 스케일링 확인

### 4. 실험 결과 (Key Results)
- **단일 코어 성능:**
  - 4KB random read: **10M+ IOPS per core**
  - 이는 I/O 요청당 약 **200-300 CPU 사이클**에 해당
  - 커널 NVMe 드라이버 대비 약 **10배 이상** 효율적
- **시스템 전체 성능:**
  - 12-14개 코어 사용 시 **120M IOPS** 달성
  - **거의 선형적 스케일링** (코어 수 증가에 비례)
  - NUMA 노드 간 I/O 시 약간의 성능 저하 (cross-NUMA penalty)
- **비교 (Linux 커널 스택):**
  - 동일 하드웨어에서 Linux 커널 NVMe + fio: 수천만 IOPS 수준
  - SPDK는 커널 대비 **수 배 높은 전체 IOPS** 달성
- **병목 분석:**
  - SSD 개수가 충분하면 **CPU가 유일한 병목**
  - PCIe 대역폭은 4KB I/O 기준으로는 병목이 되지 않음
  - NUMA topology에 맞는 SSD-코어 매핑이 중요

### 5. 한계점 및 향후 연구
- **합성 벤치마크:** `spdk_perf`를 사용한 최적 시나리오 결과이며, 실제 애플리케이션 워크로드와 다름
- **CPU 전용 사용:** SPDK 폴링에 CPU 코어를 전용으로 할당해야 하며, 다른 작업에 사용 불가
  - 120M IOPS 달성 시 12-14개 코어가 I/O 전용으로 소모
- **4KB random read만 집중:** 다른 I/O 패턴(sequential, write, mixed)에서는 다른 결과 가능
- **운영 복잡성:** SPDK 환경 설정, SSD unbinding, hugepages 설정 등 운영 부담
- **멀티테넌트 불가:** 커널 바이패스로 인해 여러 애플리케이션 간 SSD 공유 어려움
- **비용 대비 효과:** 24+ NVMe SSD를 장착한 2U 서버는 상당한 비용
- **향후:** PCIe Gen5/CXL 환경에서의 추가 스케일링 가능성, SPDK vhost를 통한 VM 환경 지원
