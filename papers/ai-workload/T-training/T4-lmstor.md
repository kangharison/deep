# [T4] LMStor: Storage Acceleration Design for Large Models

- **학회/연도:** World Wide Web Journal 2025
- **저자:** (중국 연구팀)
- **분류:** 트레이닝 스토리지 최적화

## 핵심 요약 (1~2문장)
LLM 트레이닝에 특화된 스토리지 프레임워크로, 완전 비동기 계층적 체크포인트와 다단계 장애 복구 전략을 제안. 체크포인트 중단을 분 단위에서 밀리초 단위로 줄이고, 복구 효율을 3~24배 향상시킨다.

## 읽기 전 질문
- "계층적(hierarchical) 체크포인트"란 어떤 계층을 의미하는가?
- 다단계 장애 복구 전략의 각 단계는 어떻게 구분되는가?
- 체크포인트 중단을 밀리초 수준으로 줄이는 핵심 메커니즘은?

## 주요 내용 정리

### 1. 문제 정의 (Problem)
- LLM 트레이닝의 3대 스토리지 병목:
  1. 체크포인트 중단(interruption)이 수 분 소요
  2. 장애 복구(recovery) 효율이 낮음
  3. 데이터 로딩 성능이 모델 크기 증가를 따라가지 못함

### 2. 제안 방법 (Approach)

**계층적 체크포인트:**
```
GPU VRAM ──[비동기 복사]──→ Host DRAM (캐시 계층)
                              │
                              ├──[이중 복제본(dual-replica)]
                              │
                              └──[비동기 쓰기]──→ NVMe SSD / 원격 스토리지
```

- **GPU→Host 비동기 복사**: CUDA 비동기 memcpy로 GPU 연산과 중첩
- **이중 복제본 캐시**: Host DRAM에 두 개의 체크포인트 버퍼를 번갈아 사용 (double buffering)
- **캐시 일관성**: 복제본 간 일관성을 보장하며 디스크 쓰기는 백그라운드

**다단계 장애 복구:**
- 레벨 1: Host DRAM 캐시에서 복구 (밀리초)
- 레벨 2: 로컬 NVMe SSD에서 복구 (초 단위)
- 레벨 3: 원격 스토리지에서 복구 (분 단위)
- 장애 유형(GPU 오류, 노드 오류, 전체 장애)에 따라 적절한 레벨 선택

### 3. 실험 결과
- 체크포인트 중단 시간: **분 → 밀리초** (수천 배 개선)
- 장애 복구 효율: 기존 대비 **3~24배** 향상
- 다양한 LLM 크기(수십억~수천억 파라미터)에서 검증

### 4. 한계점 및 향후 연구
- Host DRAM 용량이 체크포인트 크기를 수용해야 함
- 이중 복제본으로 인한 메모리 사용량 2배
- 네트워크 스토리지 계층의 대역폭이 낮으면 레벨 3 복구 시간 증가

## 다른 논문과의 관계
- 비교: [T1] DataStates-LLM (lazy 비동기), [T2] PCcheck (persistent concurrent)
- 동기 부여: [W2] 장애 빈도 실측, [W3] 체크포인트 규모 실측
- gpu-ssd 연결: 캐시 계층의 NVMe 쓰기에 GDS P2P DMA 활용하면 GPU→NVMe 직접 체크포인트 가능

## 발표 자료 활용 포인트
- 계층적 체크포인트 아키텍처 다이어그램
- "분 → 밀리초" 체크포인트 중단 시간 개선 수치

## 메모
- T1~T4를 함께 보면 체크포인트 최적화의 발전 방향이 선명: 동기 → 비동기(T1) → 병렬(T2) → 계층적(T4)
- 이중 복제본 + 다단계 복구는 RAID/저널링과 유사한 reliability 사고방식을 체크포인트에 적용한 것
