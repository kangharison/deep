# [W3] LLM Training in Practice: Insights from 85,000 Checkpoints

- **학회/연도:** PDSW 2025 (SC25 워크숍)
- **저자:** Glenn K. Lockwood (VAST Data, Inc.)
- **분류:** 워크로드 특성화

## 핵심 요약 (1~2문장)
18개 AI 클라우드 클러스터에서 40개 프로덕션 LLM 트레이닝 작업(45B~1T+ 파라미터)의 85,000개 이상 체크포인트를 분석한 실측 연구. Trillion-parameter 모델에서도 글로벌 체크포인트 대역폭이 **1 TB/s 미만**이면 충분하며, 실제 병목은 대역폭이 아니라 체크포인트 overlap과 비동기 계층적 전략에 있음을 실증한다.

## 읽기 전 질문
- 프로덕션 환경에서 체크포인트의 실제 크기와 빈도는 어느 정도인가?
- Trillion-parameter 모델의 체크포인트에 필요한 실제 스토리지 대역폭은?
- 벤더들이 제시하는 스토리지 요구량이 과대추정인 이유는?

## 주요 내용 정리

### 1. 분석 규모

| 항목 | 수치 |
|------|------|
| **클러스터 수** | 18개 AI 클라우드 프로바이더 |
| **트레이닝 작업** | 40개 (모두 **1,024 GPU 이상** 규모) |
| **체크포인트 수** | **85,000개 이상** |
| **모델 크기 범위** | 45B ~ **1T+ 파라미터** |
| **최대 클러스터** | 16,000 H100 GPU |

### 2. 체크포인트 크기 모델

**크기 추정 공식:**
```
체크포인트 크기 = 파라미터 수 × 16 bytes
(모델 파라미터 FP16 + Adam 옵티마이저 상태 FP32: momentum + variance)
```

**구체적 예시:**
- 1T 파라미터 모델: 단일 데이터 병렬 레플리카 기준 **~16 TB**
- 405B 모델 / 16,000 H100 GPU: 분산 저장으로 GPU당 크기는 훨씬 작음

### 3. 대역폭 측정 결과

**핵심 발견: "대역폭이 아니라 overlap이 중요하다"**

| 시나리오 | 필요 대역폭 | 비고 |
|---------|:----------:|------|
| 3,072 GPU, 1T 파라미터 | **273 GB/s** | 99.7% forward progress 달성 (손실 0.3%) |
| 가장 큰 모델 | **수백 GB/s 미만** | 1 TB/s를 넘지 않음 |
| 전체 관측 | **1 TB/s 미만이면 충분** | 벤더 가이던스 과대추정 확인 |

**대역폭이 modest한 이유:**
- 데이터 병렬: 레플리카 중 하나만 저장 → GPU당 실제 쓰기량 감소
- 체크포인트 주기가 수십 분 → 대역폭 = 크기/주기가 생각보다 작음
- GPU 수 증가 → GPU당 체크포인트 크기 오히려 감소 (분산 파티셔닝)

### 4. 비동기 계층적 체크포인팅 실태

```
프로덕션 실제 전략:

  GPU ──[빠른 비동기 복사]──→ 노드-로컬 SSD (빈번, 수 분 간격)
                                    │
                                    └──[백그라운드 드레인]──→ 글로벌 공유 스토리지 (낮은 빈도)

  핵심: 글로벌 스토리지가 보는 I/O는 GPU idle 시간과 무관
        노드-로컬 → 글로벌 드레인 시간이 진짜 병목
```

**405B 모델 / 16,000 H100 GPU:**
- MTTI (Mean Time To Interrupt): **~150분**
- 합리적 체크포인트 간격: MTTI의 1/10 = **~15분**
- **10% overlap이면 충분**: 체크포인트 쓰기 시간이 체크포인트 간격의 10% 이내

### 5. 수요측 모델 (Demand-Side Model)

논문이 제안하는 간단한 수식:
- GPU 스케일(수)과 시스템 신뢰성(MTTI)을 입력으로
- 필요한 글로벌 체크포인트 대역폭을 산출
- **벤더 가이던스가 I/O 성능 필요량을 과대평가**하는 경향을 실증적으로 입증

### 6. 핵심 발견 요약

1. **글로벌 스토리지 대역폭 요구: Trillion-parameter에서도 1 TB/s 미만**
2. **GB/s보다 overlap이 중요**: 비동기 체크포인트의 overlap ratio가 핵심 메트릭
3. **대역폭은 모델 크기에 비례하지 않음**: 비동기 계층적 전략 사용 시
4. **노드-로컬 SSD가 핵심 역할**: 글로벌 파일시스템이 아닌 로컬 SSD가 실질적 체크포인트 계층
