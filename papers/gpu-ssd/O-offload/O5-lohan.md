# [O5] LoHan: Low-Cost High-Performance Framework for LLM Fine-tuning

- **학회/연도:** 2024
- **저자:**
- **분류:** GPU-NVMe 오프로드

## 핵심 요약 (1~2문장)
CPU DRAM과 NVMe SSD를 혼합 활용하여 activation 오프로드를 최적화하는 저비용 LLM 파인튜닝 프레임워크. 텐서별 접근 빈도와 크기를 분석하여 CPU DRAM과 SSD 사이에 최적 배치하고, 비동기 전송으로 오프로드 오버헤드를 최소화.

## 읽기 전 질문
- CPU DRAM과 NVMe SSD 사이에서 텐서를 어떤 기준으로 배치하는가?
- "Low-Cost"의 의미는 하드웨어 비용인가, 성능 오버헤드인가, 아니면 둘 다인가?
- SSD만 사용하는 SSDTrain과 비교하여 DRAM+SSD 혼합의 이점은 얼마나 큰가?

## 주요 내용 정리

### 1. 문제 정의 (Problem)
- LLM 파인튜닝 시 activation 메모리가 GPU VRAM을 초과
- **SSD만 사용하는 오프로드**: 대역폭 부족으로 성능 저하가 큼
- **CPU DRAM만 사용하는 오프로드**: DRAM 용량이 제한적, 비용 높음 (DRAM $/GB >> SSD $/GB)
- 두 저장소를 단순히 계층적으로 사용하면 최적이 아님
  - 모든 텐서가 동일한 접근 패턴을 갖지 않음
  - 일부 텐서는 빠른 DRAM에, 일부는 저렴한 SSD에 배치하는 것이 유리
- 비용 효율적인 메모리 확장 방안 필요

### 2. 제안 방법 (Approach)
- **Hybrid Offload Policy**: 텐서를 CPU DRAM과 NVMe SSD에 분산 배치
  - 접근 빈도가 높고 크기가 작은 텐서 → CPU DRAM
  - 접근 빈도가 낮고 크기가 큰 텐서 → NVMe SSD
- **Cost-Aware Placement**: 텐서별 오프로드 비용(전송 시간 × 접근 횟수) 계산
- **비동기 프리페칭**: backward pass에서 필요한 activation을 미리 로드
- **압축(Compression) 기법**: activation 텐서를 압축하여 I/O 양 감소
- **메모리 풀링**: CPU DRAM의 빈 공간을 동적으로 할당/해제

### 3. 핵심 아키텍처/설계

```
  LoHan 하이브리드 오프로드 아키텍처:
  ┌─────────────────────────────────────────────┐
  │             Placement Optimizer              │
  │  ┌────────────────────────────────────────┐  │
  │  │  텐서 프로파일링 결과:                 │  │
  │  │  - 크기(size)                          │  │
  │  │  - 생존시간(lifetime)                  │  │
  │  │  - 접근빈도(access frequency)          │  │
  │  └──────────────┬─────────────────────────┘  │
  │                 │  배치 결정                  │
  │         ┌───────┴───────┐                    │
  │         ▼               ▼                    │
  │    Hot Tensors     Cold Tensors              │
  └─────┬───────────────────┬────────────────────┘
        │                   │
  ┌─────▼──────┐      ┌────▼─────┐
  │  GPU VRAM  │      │          │
  │ (활성 연산)│      │          │
  └─────┬──────┘      │          │
        │ PCIe        │          │
  ┌─────▼──────┐      │          │
  │  CPU DRAM  │      │          │
  │  (Hot 캐시)│      │          │
  │  - 작은 텐서│     │          │
  │  - 빈번 접근│     │          │
  └─────┬──────┘      │          │
        │ PCIe/DMA    │          │
  ┌─────▼─────────────▼──────────┐
  │        NVMe SSD              │
  │  (Cold 스토리지)             │
  │  - 큰 텐서                   │
  │  - 낮은 접근 빈도            │
  │  - 압축 저장                 │
  └──────────────────────────────┘

  비용 모델:
  Cost(tensor_i) = transfer_time(i) × access_count(i)
  → Cost 낮은 순서로 SSD 배치, 높은 순서로 DRAM 배치
```

### 4. 실험 결과 (Key Results)
- LLaMA, GPT-2 등 모델에서 파인튜닝 실험
- SSD-only 오프로드(SSDTrain 방식) 대비 **20~40% 학습 속도 향상**
- DRAM-only 오프로드 대비 **하드웨어 비용 50~70% 절감** (DRAM 용량 감소)
- 동일한 총 비용에서 **배치 크기를 2~3배** 늘릴 수 있음
- Activation 압축으로 I/O 양 **30~50% 추가 감소**
- GPU 메모리 사용량 **최대 60%** 절감

### 5. 한계점 및 향후 연구
- 최적 배치 결정을 위한 프로파일링 오버헤드 존재
- 모델 구조에 따라 최적 DRAM:SSD 비율이 달라져 범용적 가이드라인 부족
- 분산 학습 환경에서의 DRAM/SSD 혼합 오프로드 확장 미검증
- 압축 기법의 정확도 영향에 대한 분석 제한적
- PCIe 대역폭을 GPU-CPU와 GPU-SSD가 경쟁하는 문제 미해결
