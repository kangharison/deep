# [P3] FlashGPU: Placing New Flash Next to GPU Cores

- **학회/연도:** DAC 2019
- **저자:** Zhang et al.
- **분류:** 선행 연구 (Pre-2023)

## 핵심 요약 (1~2문장)
GPU 코어 옆에 플래시 스토리지를 직접 배치하는 아키텍처 수준 연구. BaM이 소프트웨어적으로 구현한 것을 하드웨어 관점에서 탐구.

## 읽기 전 질문
- 플래시를 GPU 패키지 내부에 배치하면 PCIe 병목이 어떻게 해소되는가?
- GPU 내부 인터커넥트 대역폭으로 플래시 접근이 충분히 빠른가?
- 실제 제조 가능성과 열/전력 제약은 어떠한가?

## 주요 내용 정리

### 1. 문제 정의 (Problem)
- GPU와 SSD가 PCIe 버스로 연결되어 있어 데이터 전송 시 PCIe 대역폭과 레이턴시가 병목
- NVMMU [P2]가 GPU-SSD 직접 접근을 제안했지만, 물리적 거리로 인한 레이턴시 문제는 해결 못함
- GPU 메모리(HBM) 용량 한계로 대규모 데이터셋 처리 시 빈번한 데이터 이동 필요
- 기존 GPU-스토리지 아키텍처는 데이터 집약적 애플리케이션에 비효율적

### 2. 제안 방법 (Approach)
- **FlashGPU**: 3D NAND 플래시를 GPU 패키지 내부(또는 인터포저 위)에 직접 배치하는 아키텍처
- GPU 내부 인터커넥트(NoC)를 통해 플래시에 접근하여 PCIe 오버헤드 제거
- GPU 메모리 계층에 플래시를 새로운 레벨로 추가: Register → Shared Mem → L1/L2 → Global Mem (HBM) → **Flash**
- 플래시 전용 컨트롤러를 GPU 내부에 통합

```
+------------------------------------------+
|              GPU Package                  |
|                                           |
|  +-------+  +-------+  +-------+         |
|  | SM 0  |  | SM 1  |  | SM N  |         |
|  +---+---+  +---+---+  +---+---+         |
|      |          |          |              |
|  +---v----------v----------v---+          |
|  |     GPU Internal NoC        |          |
|  +---+----------+----------+--+          |
|      |          |          |              |
|  +---v---+  +---v---+  +--v------+       |
|  | HBM   |  | HBM   |  | Flash   |       |
|  | Stack  |  | Stack  |  | Array   |       |
|  +-------+  +-------+  | + FTL   |       |
|                         +---------+       |
+------------------------------------------+
```

### 3. 핵심 아키텍처/설계
- **인패키지 플래시**: 3D NAND를 GPU 인터포저 위에 배치하여 HBM과 유사한 물리적 근접성 확보
- **경량 FTL**: GPU 내장 플래시 전용으로 설계된 간소화된 Flash Translation Layer
- **메모리 계층 확장**: GPU 메모리 계층에 플래시를 최하위 레벨로 추가
- **데이터 배치 정책**: 핫 데이터는 HBM, 콜드 데이터는 플래시에 자동 배치
- **채널 레벨 병렬성**: 다수의 플래시 채널을 GPU 스레드에 매핑하여 병렬 접근

### 4. 실험 결과 (Key Results)
- 시뮬레이션 기반 평가 (GPGPU-Sim 확장)
- PCIe 연결 SSD 대비 **레이턴시 약 4~10x 감소** (인터커넥트 레이턴시만 발생)
- GPU 메모리 용량을 **수 TB 수준으로 확장** 가능
- 그래프 처리, 데이터베이스 워크로드에서 기존 GPU+외장 SSD 대비 유의미한 성능 향상
- 전력 효율도 개선 (PCIe PHY 전력 소모 제거)

### 5. 한계점 및 향후 연구
- **시뮬레이션 전용**: 실제 하드웨어 프로토타입이 아닌 시뮬레이션 결과
- **제조 복잡성**: GPU 패키지에 플래시를 통합하는 것은 패키징, 열 관리 측면에서 도전적
- **플래시 수명**: GPU의 높은 쓰기 빈도가 플래시 wear-out을 가속화할 수 있음
- **비용**: GPU 패키지 면적 증가로 제조 비용 상승
- **소프트웨어 스택 미비**: 하드웨어 아키텍처 제안에 집중하여 소프트웨어 스택은 깊이 다루지 않음
- 향후: BaM [I1]이 기존 하드웨어에서 소프트웨어적으로 유사한 목표를 달성

## 다른 논문과의 관계
- 선행 연구: NVMMU [P2] (GPU-SSD 직접 접근의 하드웨어 개념)
- 후속 연구: BaM [I1] (소프트웨어적 접근으로 기존 하드웨어에서 GPU-SSD 직접 접근 구현)
- 비교 대상: GPU + 외장 NVMe SSD (PCIe 연결) 구성
